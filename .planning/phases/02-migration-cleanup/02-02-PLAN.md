3. Category filter `<select>` (lines 143-158) — replace with cfp Dropdown:
   ```tsx
   const filterOptions: DropdownOption[] = [
     { value: "", text: "All" },
     ...categoryIds.map((catId) => ({
       value: catId,
       text: schemas[catId]?.categoryLabel ?? catId,
     })),
   ];
   <Dropdown
     value={categoryFilter || null}
     options={filterOptions}
     onValueChange={(val) => setCategoryFilter(val)}
     placeholder="All"
   />
   ```
   Note: When user selects "All" (value ""), the Dropdown `onValueChange` fires `""`. This correctly resets categoryFilter. BUT cfp Dropdown treats `null` value as "show placeholder". So when `categoryFilter` is `""`, we pass `null` to show the placeholder "All". This is correct behavior.

   Actually, re-check: the "All" option has `value: ""`. When selected, `onValueChange("")` fires. Then `categoryFilter` = `""`, and `value={categoryFilter || null}` = `null`, showing placeholder. But the user selected "All" explicitly — this should show "All" as selected, not placeholder.

   Fix: Use a sentinel value instead of empty string for "All":
   ```tsx
   const ALL = "__all__";
   const filterOptions: DropdownOption[] = [
     { value: ALL, text: "All" },
     ...categoryIds.map(...)
   ];
   <Dropdown
     value={categoryFilter || ALL}
     options={filterOptions}
     onValueChange={(val) => setCategoryFilter(val === ALL ? "" : val)}
   />
   ```
   This way "All" shows as a proper selected option.
