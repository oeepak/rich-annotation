## Directory Purposes

**`src/plugin/`:**
- Purpose: Figma plugin code running in main thread
- Contains: Plugin initialization, Figma API calls, schema/annotation management
- Key files: `code.ts` (single file, ~300 lines)

**`src/ui/`:**
- Purpose: React/Preact UI running in iframe
- Contains: Components, hooks, styles
- Key files: `App.tsx` (state management), `main.tsx` (entry point)

**`src/ui/components/`:**
- Purpose: Reusable UI components
- Contains: Tab views, form inputs, data display components
- Key files:
  - `OverviewTab.tsx` - List all annotations on page with search/filter
  - `SelectedTab.tsx` - Edit annotation on selected node
  - `SchemaTab.tsx` - Edit field schemas for categories

**`src/ui/hooks/`:**
- Purpose: Custom Preact hooks
- Contains: State management, derived computation
- Key files: `useSelection.ts` - Manages form field state during editing

**`src/shared/`:**
- Purpose: Business logic shared between plugin and UI
- Contains: Types, parsing, building, validation
- No external dependencies; pure utility layer
- Used by both `src/plugin/` and `src/ui/`

**`tests/`:**
- Purpose: Unit and component tests
- Contains: Test files mirroring source structure
- Key: Tests run with `npm test` (Vitest)

**`docs/`:**
- Purpose: Project documentation
- Contains: Planning documents, phase descriptions

**`.planning/codebase/`:**
- Purpose: Codebase analysis documents (ARCHITECTURE.md, STRUCTURE.md, etc.)
- Generated by GSD mapping commands

## Key File Locations

**Entry Points:**
- `src/plugin/code.ts`: Plugin code entry point (Figma main thread)
- `src/ui/main.tsx`: UI entry point (iframe root)
- `src/ui/App.tsx`: Application root component

**Configuration:**
- `manifest.json`: Figma plugin metadata (id, name, permissions)
- `tsconfig.json`: TypeScript compiler settings
- `vitest.config.ts`: Test runner configuration
- `package.json`: Dependencies and build scripts

**Core Logic:**
- `src/shared/parseText.ts`: Parse annotation text into fields
- `src/shared/buildText.ts`: Build field values into annotation text
- `src/shared/validateField.ts`: Validate field type and value
- `src/shared/types.ts`: All type definitions

**State Management:**
- `src/ui/App.tsx`: Global view state (selected tab, node selection)
- `src/ui/hooks/useSelection.ts`: Form field state during editing
- `src/plugin/code.ts`: Schema persistence and selection tracking

**Component Views:**
- `src/ui/components/OverviewTab.tsx`: Overview of all page annotations
- `src/ui/components/SelectedTab.tsx`: Editor for selected node annotation
- `src/ui/components/SchemaTab.tsx`: Schema definition editor
- `src/ui/components/FieldInput.tsx`: Individual field input component

**Testing:**
- `tests/setup.ts`: Vitest environment setup
- `tests/*.test.ts`: Shared utility tests
- `tests/ui/*.test.tsx`: Component tests

## Naming Conventions

**Files:**
- Camel case with `.tsx` for components: `OverviewTab.tsx`, `FieldInput.tsx`
- Camel case with `.ts` for utilities: `buildText.ts`, `validateField.ts`
- Camel case with `.test.tsx` / `.test.ts` for tests
- Config files lowercase: `tsconfig.json`, `vitest.config.ts`, `manifest.json`

**Directories:**
- Lowercase kebab or plural: `components/`, `hooks/`, `shared/`, `tests/`, `docs/`
- PascalCase reserved for monorepo packages (not used here)

**Functions:**
- camelCase: `buildText()`, `parseText()`, `validateField()`
- Hooks start with `use`: `useSelection()`, `usePluginMessage()`
- Handlers end with `-Handler`: interfaces like `SelectionChangedHandler`

**Types:**
- PascalCase: `AnnotationInfo`, `FieldSchema`, `ParsedField`, `SchemaStore`
- Union types lowercase: `"matched"`, `"not_matched"`, `"no_schema"` (ParseMatch)
- Group names: `FieldData`, `GroupValues` (internal)

**CSS Classes:**
- kebab-case: `.plugin-container`, `.tab-content`, `.action-bar`
- Utility pattern: `.btn`, `.btn-primary`, `.btn-secondary`

## Where to Add New Code

**New UI Component:**
- Location: `src/ui/components/[ComponentName].tsx`
- Pattern: Functional component with props interface
- Import shared types from `@shared/types`
- Example: `src/ui/components/OverviewTab.tsx` (180 lines)

**New Shared Utility:**
- Location: `src/shared/[utilityName].ts`
- Pattern: Export named functions, no dependencies on UI/plugin code
- Add corresponding test: `tests/[utilityName].test.ts`
- Example: `src/shared/buildText.ts` + `tests/buildText.test.ts`

**New Hook:**
- Location: `src/ui/hooks/[useHookName].ts`
- Pattern: Export function starting with `use`
- Import from `preact/hooks` not `react`
- Add test: `tests/ui/[useHookName].test.tsx`
- Example: `src/ui/hooks/useSelection.ts` (104 lines)

**New Tab View:**
- Location: `src/ui/components/[TabName]Tab.tsx`
- Pattern: Accept props from `App.tsx`, send messages via `postToPlugin()`
- Update routing in `src/ui/App.tsx` switch statement
- Example: `src/ui/components/SchemaTab.tsx`

**New Plugin Handler:**
- Location: Add message handler in `src/plugin/code.ts`
- Add type definition in `src/shared/messages.ts`
- Update `PluginMessage` type union
- Example: Handlers at lines 219-295 in code.ts

**New Test:**
- Location: `tests/` mirroring source structure
- Naming: `[name].test.ts` or `[name].test.tsx`
- Pattern: Use vitest + @testing-library/preact
- Setup: Import from `vitest` and `@testing-library/preact`

## Special Directories

- Purpose: CSS files for UI styling
