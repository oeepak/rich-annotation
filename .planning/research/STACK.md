## Critical: JSX Configuration

### The Conflict

The current project tsconfig uses:
```json
"jsx": "react-jsx",
"jsxImportSource": "preact"
```

This is WRONG for cfp. The cfp build (esbuild) hardcodes:
```js
jsxFactory: 'h',
jsxFragment: 'Fragment',
```

This means esbuild expects the **classic JSX transform** (`h(...)` calls), not the automatic import source transform. The correct tsconfig for cfp is:

```json
{
  "compilerOptions": {
    "jsx": "react",
    "jsxFactory": "h",
    "lib": ["DOM", "ES2020"],
    "module": "ES2020",
    "moduleResolution": "Node",
    "strict": true,
    "esModuleInterop": true,
    "isolatedModules": true,
    "skipLibCheck": true,
    "typeRoots": ["node_modules/@figma", "node_modules/@types"]
  },
  "include": ["src/**/*.ts", "src/**/*.tsx"]
}
```

Source: official `@create-figma-plugin/tsconfig` base config (verified from cfp repository + build source).

### Consequence for `h` Import

Because `jsxFactory: 'h'` is used, **every `.tsx` file must explicitly import `h`**:

```tsx
import { h } from 'preact'
// or
import { h, Fragment } from 'preact'
```

Without this import, TypeScript emits `h(...)` calls that reference an undefined `h`, causing runtime errors.

### Consequence for Hooks

Hooks must be imported from `'preact/hooks'`, not `'react'`:

```tsx
// WRONG
import React, { useState, useCallback } from 'react'

// CORRECT
import { h } from 'preact'
import { useState, useCallback } from 'preact/hooks'
```

## Import Patterns

### UI Entry Point (src/ui/main.tsx)

The current file is already correct cfp pattern:

```tsx
import { render } from '@create-figma-plugin/ui'
import { h } from 'preact'
import { App } from './App'

function Plugin() {
  return <App />
}

export default render(Plugin)
```

`render` wraps the component and returns a function that cfp's build system calls with `(rootNode, props)`. The `export default` is required.

### Plugin Entry Point (src/plugin/code.ts)

Already correct cfp pattern. The `export default async function()` signature is what cfp's entry file generator wraps:

```ts
import { emit, on, showUI } from '@create-figma-plugin/utilities'

export default async function () {
  showUI({ width: 360, height: 560 })
  on<SomeHandler>('EVENT_NAME', (data) => { ... })
  emit<AnotherHandler>('ANOTHER_EVENT', data)
}
```

### Messaging: emit/on Pattern

**Plugin → UI:**
```ts
// In code.ts (plugin context)
import { emit } from '@create-figma-plugin/utilities'
import type { SelectionChangedHandler } from '../shared/messages'

emit<SelectionChangedHandler>('SELECTION_CHANGED', { nodeId, nodeName, ... })
```

**UI → Plugin:**
```tsx
// In any UI component
import { emit } from '@create-figma-plugin/utilities'
import type { GetAnnotationsHandler } from '@shared/messages'

emit<GetAnnotationsHandler>('GET_ANNOTATIONS')
```

**UI listening to plugin events:**
```tsx
import { on } from '@create-figma-plugin/utilities'
import { useEffect } from 'preact/hooks'

useEffect(() => {
  return on<SelectionChangedHandler>('SELECTION_CHANGED', (data) => {
    // handle data
  })
}, [])
```

`on()` returns a deregistration function — return it from `useEffect` for cleanup.

### Handler Type Definition (src/shared/messages.ts)

Already correct. Use `EventHandler` from `@create-figma-plugin/utilities`:

```ts
import { EventHandler } from '@create-figma-plugin/utilities'

export interface SelectionChangedHandler extends EventHandler {
  name: 'SELECTION_CHANGED'
  handler: (data: { nodeId: string | null; ... }) => void
}
```

### Missing: usePluginMessage Hook

`usePluginMessage` does not exist in cfp. It was a custom hook that is now missing (the file `src/ui/hooks/usePluginMessage.ts` was deleted mid-migration). Replace with direct `on()` calls in `useEffect`.

The `App.tsx` currently imports `{ usePluginMessage, postToPlugin }` from this missing file. Replace:
- `usePluginMessage(handleMessage)` → multiple `on<Handler>()` calls in useEffect
- `postToPlugin({ type: 'X', ...data })` → `emit<XHandler>('X', data)`

Also: `PluginMessage` union type in messages.ts does not exist and is not needed in cfp pattern. Remove that import.

## UI Component Library

All components import directly from `'@create-figma-plugin/ui'`. Never use native HTML form elements when a cfp equivalent exists — they handle Figma-native event types and styling automatically.

### Available Components (verified from package exports)

**Form Controls:**
- `Button` — primary/secondary variant via `secondary` prop
- `Checkbox`
- `Dropdown` — replaces `<select>`. Uses `DropdownOption[]` array prop.
- `TextboxNumeric` — for numbers. Separate `value` (string) and `onNumericValueInput` (number|null) props.
- `Textbox` — single-line text. Use `value` + `onValueInput` props.
- `TextboxMultiline` — multi-line text.
- `TextboxAutocomplete`
- `Toggle`
- `RadioButtons`
- `RangeSlider`
- `SegmentedControl`
- `SearchTextbox`

**Layout:**
- `Container` — padding wrapper. `space` prop: `'small' | 'medium' | 'large' | 'extraLarge'`
- `VerticalSpace` — spacer. `space` prop: `'extraSmall' | 'small' | 'medium' | 'large' | 'extraLarge'`
- `Columns` — horizontal layout. `space` prop for gap.
- `Stack` — vertical stack.
- `Inline` — inline elements.
- `MiddleAlign` — vertical center alignment.

**Typography:**
- `Text` — base text element
- `Bold`, `Muted`, `Code`, `Link` — inline text modifiers
- `Divider`
- `Banner` — status/alert messages

**Interactive:**
- `IconButton`
- `Tabs` + `TabsOption[]`
- `Modal`
- `Disclosure`
- `SelectableItem`
- `Layer`

**Hooks:**
- `useWindowResize` — for resizable plugin windows
- `useForm` — form state management
- `useInitialFocus` — focus management
- `useFocusTrap`, `useMouseDownOutside`, `useScrollableMenu`, `useWindowKeyDown`

### Component Usage Pattern (verified from preact-rectangles template)

```tsx
import {
  Button,
  Columns,
  Container,
  Dropdown,
  Text,
  TextboxNumeric,
  VerticalSpace
} from '@create-figma-plugin/ui'
import { emit } from '@create-figma-plugin/utilities'
import { h } from 'preact'
import { useState } from 'preact/hooks'

function MyComponent() {
  const [value, setValue] = useState('')
  return (
    <Container space="medium">
      <VerticalSpace space="small" />
      <Text>Label</Text>
      <VerticalSpace space="extraSmall" />
      <Textbox value={value} onValueInput={setValue} />
      <VerticalSpace space="medium" />
      <Button onClick={() => emit<SomeHandler>('EVENT')}>
        Apply
      </Button>
    </Container>
  )
}
```

### Dropdown vs select

Native `<select>` does not render with Figma styling. Use cfp `Dropdown`:

```tsx
import { Dropdown, DropdownOption } from '@create-figma-plugin/ui'

const options: DropdownOption[] = [
  { value: 'a', text: 'Option A' },
  { value: 'b', text: 'Option B' },
  { separator: true },  // divider
  { header: 'Group' }   // section header
]

<Dropdown
  options={options}
  value={selected}
  onChange={(e) => setSelected(e.currentTarget.value)}
/>
```

### Textbox vs input

Native `<input>` does not render with Figma styling. Use cfp `Textbox`:

```tsx
import { Textbox } from '@create-figma-plugin/ui'

<Textbox
  value={text}
  onValueInput={setText}  // receives string directly, not event
  placeholder="Enter value..."
/>
```

Note: cfp components use `onValueInput` (receives value directly), not `onChange` (receives event).

## What NOT to Use

| Avoid | Why | Use Instead |
|-------|-----|-------------|
| `import React from 'react'` | `react` package not installed; TypeScript type-checks fail. cfp aliases react→preact/compat at esbuild level only, not tsc level. | `import { h } from 'preact'` |
| `import { useState } from 'react'` | Same — React types not installed | `import { useState } from 'preact/hooks'` |
| `"jsx": "react-jsx"` in tsconfig | Conflicts with cfp esbuild jsxFactory: 'h'. Generates `jsx/jsx-runtime` calls instead of `h()` calls. | `"jsx": "react"` + `"jsxFactory": "h"` |
| `"jsxImportSource": "preact"` in tsconfig | Generates automatic import of `preact/jsx-runtime`, but cfp build does not configure esbuild for this transform. | Remove this option |
| `postToPlugin({ type: 'X', data })` | Custom wrapper that doesn't exist; the hook file was deleted. | `emit<XHandler>('X', data)` |
| `usePluginMessage(handler)` | Custom hook that doesn't exist; file deleted mid-migration. | `on<Handler>('EVENT', handler)` in `useEffect` |
| `PluginMessage` union type | Not a cfp pattern; messages.ts doesn't export it and doesn't need to. | Individual typed `EventHandler` interfaces |
| Editing manifest.json directly | Overwritten on every build run. | Configure via package.json `"figma-plugin"` key |
| Raw `<input>`, `<select>`, `<button>` | No Figma-native styling, inconsistent appearance. | cfp `Textbox`, `Dropdown`, `Button` components |
| `figma.ui.postMessage()` / `window.onmessage` | Low-level Figma API; no type safety. | `emit<Handler>()` / `on<Handler>()` from cfp utilities |

## Alternatives Considered

| Recommended | Alternative | When to Use Alternative |
|-------------|-------------|-------------------------|
| `emit`/`on` from cfp utilities | Raw `figma.ui.postMessage` / `window.onmessage` | Never — cfp utilities provide type safety and are designed for this |
| cfp `Dropdown` | Native `<select>` | Never in cfp context — no Figma-native styling |
| `import { h } from 'preact'` per file | `"jsxImportSource": "preact"` auto-import | If you abandon cfp build for Vite. In cfp context, explicit `h` import is required. |
| `on()` return value for cleanup | Manual `off()` or ref tracking | `on()` returns a deregister function; always return it from useEffect. |
